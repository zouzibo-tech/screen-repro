# API 速率限制配置

> **说明**：本文件定义当前使用的AI模型的速率限制参数。
> 子agent派发时会读取此文件，确保不超过API限制。
> 不同模型有不同的限制，请根据实际情况填写。

---

## 当前模型

**默认配置**: mimo / mimo-v2.5（工作台模型）
**RPM**: 100 | **TPM**: 10M
**配置日期**: 2026-06-02

---

## 速率限制参数

| 参数 | 值 | 说明 |
|------|-----|------|
| **最大并发请求数** | 1 | 同时运行的子agent数量 |
| **每分钟最大请求数（RPM）** | 100 | API每分钟允许的请求次数 |
| **每分钟最大输入Token数** | 10,000,000 | 每分钟允许的输入token总量 |
| **每分钟最大输出Token数** | 10,000,000 | 每分钟允许的输出token总量 |
| **请求间隔（秒）** | 3 | 两次请求之间的最小间隔 |

---

## 429错误处理策略

| 策略 | 说明 |
|------|------|
| **自动重试** | 收到429后，自动退避重试 |
| **最大重试次数** | 4次，超过后标记该文献为"ERROR"并跳过 |
| **退避策略** | 30秒 → 60秒 → 120秒 → 240秒（指数退避） |
| **批量间隔** | 每处理完1篇后，等待3秒再处理下一篇 |

## 使用方式

1. 复制本模板到项目目录：`03_Screening/RATE_LIMIT.md`
2. 填入当前使用的模型参数
